# План: поездки — таблица и интерфейс

## Цель

Иметь отдельную **таблицу поездок** с датами, **привязывать фото к поездкам** (через папку ЯД и напрямую) и **интерфейс просмотра поездки** со списком, карточкой, явными привязками и предложениями по датам.

## Источник данных по поездкам

Часть данных уже есть в таблице Excel:  
`YandexDisk\Билеты и Путешествия\Поездки.xlsx` — использовать при создании/синхронизации таблицы поездок (даты, место и т.п.).

## Задачи

### 1. Таблица поездок в БД

- Отдельная таблица поездок: id, название, даты (start_date, end_date или аналог), место (страна/город), путь к папке на ЯД (например `disk:/Фото/Путешествия/2024 Тургояк`), заглавное фото (file_id или path), создано/обновлено.
- Импорт/синхронизация из Excel (`Поездки.xlsx`) — скрипт или шаг, чтобы заполнять таблицу поездок из файла (и при необходимости из подпапок ЯД `disk:/Фото/Путешествия/`).

**Файлы:** `backend/common/db.py` (схема таблицы поездок), миграция; скрипт импорта из Excel (например в `backend/scripts/tools/`).

### 2. Привязка фото к поездкам

- **Через папку ЯД поездки:** файлы, лежащие в папке поездки на ЯД (`disk:/Фото/Путешествия/<название>/...`), считаются привязанными к этой поездке (связь по пути или по file_id после инвентаризации).
- **Напрямую:** пользователь в интерфейсе привязывает файл к поездке (запись в таблице связей file_id ↔ trip_id или использование существующей группы с привязкой к поездке).

**Файлы:** схема БД (таблица связей файл–поездка или использование `file_groups` с привязкой к trip_id), API привязки/отвязки.

### 3. Интерфейс: список поездок

- **Список поездок:** заглавное фото (обложка), даты, место. Ссылка на карточку поездки.
- Данные из таблицы поездок + при необходимости подтягивание папок с ЯД.

**Файлы:** страница/раздел «Поездки», роут, шаблон списка, API списка поездок.

### 4. Карточка поездки

- **Показать файлы, явно привязанные к поездке** (через папку ЯД или прямую привязку).
- **Предлагать новые по датам:** файлы из БД, у которых дата съёмки (taken_at) попадает в диапазон поездки с **люфтом ±1 неделя**, но ещё не привязаны к поездке.
- Кнопки у предложенных: **«Привязать к поездке»** и **«Не относится к поездке»** (чтобы больше не предлагать этот файл к этой поездке или пометить отказ).

**Файлы:** шаблон карточки поездки, API: привязанные файлы, предложения по датам, действия привязать / не относится.

## Порядок выполнения

1. Таблица поездок в БД + импорт из Excel (и при необходимости синхронизация с папками ЯД).
2. Механизм привязки фото к поездкам (через папку ЯД и напрямую).
3. Интерфейс: список поездок (заглавное фото, даты, место).
4. Карточка поездки: привязанные файлы, предложения по датам (±1 нед), кнопки привязать / не относится.

## Связь с планом «Сортировка по финальным папкам»

В плане сортировки по финальным папкам предусмотрен **скрипт синхронизации поездок** (Excel + папки ЯД) для единообразия названий при раскладке в `disk:/Фото/Путешествия/`. Полный интерфейс поездок и таблица — в рамках этого плана.
